# System Architecture

## Overview

StaffSync follows the **Model-View-Controller (MVC)** architectural pattern using Spring Boot. The application is structured into three main layers that communicate sequentially.

## Architecture Diagram

```
Browser (Client)
      |
      v
  Thymeleaf Views (HTML/CSS/JS)
      |
      v
  Controllers (Spring MVC)
      |
      v
  Repositories (Spring Data JPA)
      |
      v
  PostgreSQL Database
```

## Layer Explanation

### 1. Presentation Layer (Thymeleaf Views)

- Located in `src/main/resources/templates/`
- Uses Thymeleaf templates with HTML5, CSS3, Bootstrap 5
- Server-side rendering with dynamic data binding via `th:*` attributes
- No JavaScript frameworks (vanilla JS only)

### 2. Controller Layer (Spring MVC)

- Located in `src/main/java/com/staffsync/controller/`
- Handles HTTP requests and returns view names
- Injects repositories directly (simplified layered architecture for college project)
- Manages model attributes and flash messages
- Uses Spring Security annotations for role-based access

### 3. Data Layer (Repository)

- Located in `src/main/java/com/staffsync/repository/`
- Uses Spring Data JPA interfaces extending `JpaRepository`
- Provides derived query methods and custom `@Query` annotations
- Automatically generates SQL from method names

### 4. Database Layer (PostgreSQL)

- PostgreSQL 14+ database
- Schema auto-generated by Hibernate (`ddl-auto: update`)
- 6 tables with foreign key relationships

## Spring Boot Architecture

```
Application Startup
      |
      v
  DataSeeder (CommandLineRunner)
      |-- Creates roles
      |-- Creates users
      |-- Creates employees
      |-- Seeds sample data
      |
      v
  Spring Security Filter Chain
      |-- Authentication filter
      |-- Authorization filter
      |-- CSRF protection
      |-- Session management
      |
      v
  Request Flow
      |-- Request enters Filter Chain
      |-- Authenticated? → Yes → Check Role
      |-- Role matches? → Yes → Forward to Controller
      |-- Controller processes → Queries Repository
      |-- Repository fetches from DB → Returns data
      |-- Controller adds data to Model
      |-- Thymeleaf renders HTML → Response sent to browser
```

## Role-Based Authentication

| Role     | Accessible Modules                                      |
|----------|---------------------------------------------------------|
| ADMIN    | Dashboard, Employees, Attendance, Leave, Payroll, Reports |
| EMPLOYEE | Dashboard, Attendance (own), Leave (own), Payroll (own)  |

- Authentication: Form-based login with session management
- Password Encoding: BCrypt
- CSRF Protection: Enabled globally
- Session: Single session per user

## Key Design Decisions

- **No Service Layer**: Controllers interact directly with repositories to keep the project simple and college-level
- **Server-Side Rendering**: Thymeleaf for simplicity (no React/Angular/Vue)
- **Auto Schema Generation**: Hibernate `ddl-auto: update` for zero manual SQL setup
- **Seeder Pattern**: `CommandLineRunner` for demo data on fresh databases
